# How to Contribute

Token World thrives on collaboration and contributions from developers, researchers, and enthusiasts. Whether you’re fixing bugs, suggesting new features, or improving the documentation, your input is valuable to the project.

---

## Ways to Contribute

1. **Report Issues**
   - Found a bug or unexpected behavior? Help us improve by reporting it on the [Issues page](https://github.com/reubenjohn/token-world/issues).
   - Include:
     - A clear title and description of the issue.
     - Steps to reproduce the problem.
     - Your system details (e.g., OS, Python version, hardware).
     - Logs or screenshots, if applicable.

2. **Suggest Features**
   - Have a great idea for a new feature or enhancement? Start a conversation on the [Discussions page](https://github.com/reubenjohn/token-world/discussions).
   - Include:
     - A detailed description of your idea.
     - Potential use cases and benefits.
     - Suggestions for implementation, if possible.

3. **Fix Bugs**
   - Check the [Issues page](https://github.com/reubenjohn/token-world/issues) for bugs labeled `good first issue` or `help wanted`.
   - Comment on the issue to let others know you’re working on it.

4. **Contribute Code**
   - Add new features, fix bugs, or improve existing code.
   - Follow the coding standards outlined in [Coding Standards](coding_standards.md).
   - Submit a pull request (PR) with a clear description of the changes.

5. **Improve Documentation**
   - Found outdated or unclear documentation? Update it!
   - Check for issues labeled `documentation` on the [Issues page](https://github.com/reubenjohn/token-world/issues).

---

## Getting Started with Contributions

1. **Fork the Repository**
   - Fork the [Token World repository](https://github.com/reubenjohn/token-world) to your GitHub account.
   - Clone your forked repository:
     ```bash
     git clone https://github.com/your-username/token-world.git
     ```
   - Navigate to the project directory:
     ```bash
     cd token-world
     ```

2. **Set Up the Development Environment**
   - See [README.md#setup](https://github.com/reubenjohn/token_world?tab=readme-ov-file#setup)

3. **Create a Feature Branch**
   - Always create a new branch for your contributions:
     ```bash
     git checkout -b feature/your-feature-name
     ```

4. **Make Your Changes**
   - Write clean, well-documented code.
   - Include tests for your changes where applicable.

5. **Run Tests**
   - Ensure your changes don’t break existing functionality:
     ```bash
     pytest
     ```

6. **Submit a Pull Request**
   - Push your branch to GitHub:
     ```bash
     git push origin feature/your-feature-name
     ```
   - Go to the original repository and create a pull request.
   - Provide a clear title and description of your changes.

---

## Need Help?

If you’re unsure how to start or have questions, reach out through the [Issues page](https://github.com/username/token-world/issues) or [Discussions](https://github.com/username/token-world/discussions).

We’re excited to see your contributions!
